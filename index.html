GlobOptBernstein

Mobile robots typically use a receding-horizon strategy to move through the world, where they execute an old plan while computing a new plan to incorporate new sensor information. A plan should be dynamically feasible, meaning that it obeys constraints like the robot's dynamics and obstacle avoidance; it should have liveness, meaning the robot does not stop to plan so frequently that it cannot accomplish tasks; and it should be optimal, meaning that the robot tries to satisfy a user-specified cost function such as reaching a goal location as quickly as possible. Reachability-based Trajectory Design (RTD) is a planning method that can generate provably dynamically-feasible plans. However, RTD requires solving a nonlinear, non-convex optimization program at each planning iteration, and therefore cannot make optimality guarantees; furthermore, RTD can struggle with liveness because the robot brakes to a stop when the solver finds local minima or cannot find a feasible solution. This paper proposes RTD*, which is able to certify that the robot finds the globally optimal plan (if such a plan exists) at each planning iteration. This method is enabled by a novel Parallelized Constrained Bernstein Algorithm (PCBA), which is a branch-and-bound method for polynomial optimization. The contributions of this paper are: the implementation of PCBA; proofs of bounds on the time and memory usage of PCBA; a comparison of PCBA to state of the art solvers; and the demonstration of PCBA/RTD* on a mobile robot.
